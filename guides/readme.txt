────────────────────────────────────────────────────────────────────────────────
  1. 시간 배정 (Time Assignment)
────────────────────────────────────────────────────────────────────────────────

  ■ 개요
  매 학기 또는 필요 시점에 각 팀이 동아리방을 사용할 요일/시간을 신청받고,
  관리자가 배정 결과를 확정하면 시간표에 반영됩니다.

  ■ 팀 관리 (admin.html → "팀" 탭)
  - [+ 팀 추가]로 팀 생성: 이름, 유형(합주/스쿨/이외), 색상, 소개, 구성원 입력
  - 구성원은 "이름 학번뒷4자리 세션" 형식으로 한 줄에 한 명씩 입력
    예)  홍길동 1234 보컬
  - 팀 삭제 시 해당 팀의 모든 슬롯과 신청도 함께 삭제됨 (주의)

  ■ 신청 회차 만들기 (admin.html → "시간 배정" 탭)
  1. [회차 생성] 클릭
  2. 오픈/마감 시각을 예약하거나 즉시 열기 선택
  3. [열기] 클릭 → 상태가 "신청 중"으로 전환

  ■ 신청 모니터링
  - 신청 중 상태에서 각 팀의 신청 내역이 실시간으로 표시됨
  - 한 팀이 여러 번 제출하면 가장 최근 제출만 유효
  - 마감 시각이 되면 자동으로 "신청 마감" 상태로 전환

  ■ 배정 실행
  1. 신청 마감 후 [배정 실행] 클릭
  2. 시스템이 1지망 → 2지망 → 3지망 순서로 자동 배정
     - 동일 시간에 중복 신청 시 제출 시각 빠른 팀 우선
     - 모든 지망이 선점된 팀은 미배정으로 남음
  3. 결과 화면에서 배정 내역 확인

  ■ 초안 승인 → 시간표 반영
  1. 배정 결과를 확인한 뒤 [초안 승인] 클릭
  2. 현재 시즌의 기존 base_slots가 모두 삭제되고 새 배정 결과로 교체됨
  3. 시간표(index.html)에 즉시 반영

  ■ 슬롯 직접 편집 (admin.html → "시간표" 탭)
  - 회차 배정 외에 슬롯을 직접 추가/수정/삭제 가능
  - 주차별 예외 설정: "이번 주만 휴무", "이번 주만 시간 변경", "추가 슬롯" 등

  ■ 신청/미사용 신고 처리 (우측 사이드바 또는 "요청" 탭)
  - 팀원들이 제출한 "추가 사용 신청", "미사용 신고"가 이곳에 표시됨
  - [승인] → 해당 주에 예외 슬롯 자동 생성
  - [거절] → 신청 거절 처리
  - [종료 처리] → 해당 팀의 base_slot 영구 삭제 (팀이 해체된 경우)


────────────────────────────────────────────────────────────────────────────────
  2. 스쿨 신청 (School Application)
────────────────────────────────────────────────────────────────────────────────

  ■ 개요
  악기 스쿨을 운영할 때 수강생을 모집하고 반 배정을 처리합니다.
  학생들은 1지망/2지망 반을 선택해 신청하고, 시스템이 자동으로 배정합니다.

  ■ 회차 만들기 (admin.html → "스쿨" 탭)
  1. [+ 새 스쿨 회차 만들기] 클릭
  2. 학기·회차 번호 입력 (예: 1학기 1차 → "1학기 1차 스쿨 신청"으로 자동 명명)
  3. 생성하면 "초안(draft)" 상태

  ■ 반 추가 (초안 단계에서만)
  1. [+ 반 추가] 클릭
  2. 반 이름, 담당 이름, 정원, 요일, 시간(시), 스쿨 설명 입력
  3. 여러 반 추가 가능
  - 반 정보 수정: 각 반 우측의 [수정] 버튼
  - 반 삭제: [삭제] 버튼 (연결된 신청 데이터도 함께 삭제)
  - 오픈/마감 예약 및 "이전 회차 수강생 후순위" 여부 설정 가능

  ■ 모집 열기
  - 초안 단계에서 [열기] 클릭, 또는 오픈 예약 시각에 자동 전환
  - 학생들이 school.html에서 신청 가능

  ■ 신청 처리 방식
  신규 수강생:
  - 제출 즉시 1지망 배정 시도 → 불가 시 2지망 → 불가 시 미배정

  이전 회차 수강생 (prioritize_returning = true인 경우):
  - 신청 시 "대기" 상태로 저장
  - 마감 후 일괄 처리: 제출 순서대로 1지망/2지망 배정 시도

  ■ 마감
  - [마감하기] 클릭, 또는 마감 시각에 자동 전환
  - 마감 즉시 대기 중이던 수강생들이 자동 배정됨
  - 결과: 배정됨 / 미배정 두 가지 상태

  ■ school.html에서 학생에게 표시되는 정보
  - 반 이름, 담당 이름, 요일/시간
  - [스쿨 설명 보기] 버튼으로 긴 설명 펼치기/접기 가능
  - 배정 결과(이름, 학번 뒷 3자리)


────────────────────────────────────────────────────────────────────────────────
  3. 합주 신청 (Ensemble Application)
────────────────────────────────────────────────────────────────────────────────

  ■ 개요
  합주/버스킹 공연을 위해 곡을 신청받고, 세션별로 참여자를 모집한 뒤
  DnD(드래그 앤 드롭) 방식으로 최종 팀을 구성합니다.
  일반 합주(regular)와 버스킹(busking) 두 종류가 독립적으로 운영됩니다.

  ■ 단계 구조
  2차 세션 신청 없는 경우: 1준비 → 2곡신청 → 3대기 → 4세션신청 → 5최종팀구성 → 6완료
  2차 세션 신청 있는 경우: 1준비 → 2곡신청 → 3대기 → 4세션신청 → 5대기 →
                            6 2차세션신청 → 7최종팀구성 → 8완료

  ■ 회차 만들기 (admin.html → "합주" 탭 → [회차 생성])
  설정 항목:
  - 회차 이름 (예: 2025-1 합주)
  - 최대 곡 수, 1인 최대 신청 곡 수, 1인 최대 세션 참여 곡 수
  - 곡 신청 오픈/마감 예약
  - 세션 신청 오픈/마감 예약
  - 2차 세션 신청 여부 및 설정:
    · 오픈/마감 시각
    · 신청 대상: any_song (모든 곡) / missing_only (빈 세션 있는 곡만)

  ■ 1단계: 준비 (draft)
  - 회차 생성 직후 상태
  - [설정]으로 일정 수정 가능
  - [곡 신청 즉시 열기]로 수동 전환, 또는 예약 시각에 자동 전환

  ■ 2단계: 곡 신청 (song)
  학생이 ensemble.html에서 입력:
  - 신청자 성명, 학번
  - 곡 제목, 아티스트
  - 이 곡에 필요한 세션 전체 (체크박스, 복수 선택)
  - 내가 담당할 세션 (필요 세션에 체크된 것만 선택 가능)

  제약:
  - 동일 곡명+아티스트 중복 불가 (대소문자·공백 무시)
  - 1인당 최대 신청 곡 수 초과 불가
  - 전체 신청 곡 수 초과 불가

  관리자:
  - 신청된 곡 목록 실시간 확인
  - 각 곡 [확정] / [거절] 처리
  - [이전 단계] 버튼으로 draft로 되돌리기 가능
  - 마감 시각에 자동으로 다음 단계 전환

  ■ 3단계: 대기 (song_end)
  - 곡 신청 종료 후 세션 신청 전 대기 구간
  - 세션 신청 오픈 시각 예약 시 카운트다운 표시 → 자동 전환
  - 관리자: [세션 신청 즉시 열기]로 수동 전환 가능

  ■ 4단계: 1차 세션 신청 (session)
  학생이 ensemble.html에서:
  - 확정된 곡 중 참여할 곡 선택
  - 신청 가능한 세션 선택 (신청자가 담당하는 세션 제외)
  - 1인당 최대 참여 곡 수 초과 불가
  - 동일 곡에 중복 신청 불가

  참고:
  - 곡을 신청한 사람(신청자)의 세션은 곡 신청 시 이미 등록되어 있어
    세션 신청 단계를 거치지 않아도 팀에 포함됨

  관리자:
  - 각 곡의 세션별 신청자 실시간 확인
  - 마감 시각에 자동으로 session_end로 전환

  ■ 5단계: 1차 팀 구성 (session_end)
  관리자 전용 DnD 모달:
  1. [팀 구성 시작] 클릭
  2. 왼쪽 풀(pool): 세션 미배정 인원 카드
     오른쪽: 각 곡별 팀 구성 영역
  3. 카드를 드래그해 곡으로 끌어다 놓거나 풀로 되돌리기
     - 신청자 카드는 이동 불가 (고정, 녹색 배경 "신청자" 배지)
     - 동일 곡에서 같은 세션 중복 시 주황색으로 충돌 표시
  4. [임시저장]으로 진행 상황 저장 (새로고침해도 복원)
  5. [확정]을 클릭하면:
     - 곡에 배치된 인원 → status = 'confirmed', song_id 업데이트
     - 풀에 남은 인원 → status = 'rejected'
  6. 확정 후 [2차 세션 신청으로] 또는 [완료] 버튼 표시
     - 2차 세션 신청이 설정된 경우: [2차 세션 신청으로] 클릭 시
       미확정(pending) 상태인 1차 신청 데이터 자동 삭제 후 다음 단계로

  학생 화면:
  - 팀 구성 결과(이름·세션)가 공개됨
  - rejected(미배정)된 신청은 취소선으로 표시

  ■ 6단계: 2차 세션 신청 (session2) — 선택적
  - 1차에서 부족했던 세션이나 전체 곡에 추가 세션 신청
  - missing_only 모드: 1차에서 아무도 신청 안 한 세션만 신청 가능
  - any_song 모드: 모든 확정 곡의 세션에 신청 가능
  - 신청자 담당 세션에는 여전히 신청 불가

  ■ 7단계: 최종 팀 구성 (session2_end)
  - 5단계와 동일한 DnD 방식
  - 2차 신청 카드에는 "2차" 배지 표시
  - 확정 후 [완료] 버튼으로 마지막 단계로 전환

  ■ 8단계: 완료 (closed)
  - 최종 팀 구성 결과가 ensemble.html에 공개
  - 새 회차 생성 가능

  ■ 이전 단계 되돌리기
  모든 단계(준비 제외)에서 [이전 단계] 버튼으로 한 단계씩 되돌릴 수 있음
  주의: 되돌려도 DB에 저장된 신청 데이터는 삭제되지 않음


────────────────────────────────────────────────────────────────────────────────
  4. 공지사항 / 연락처 관리 (admin.html → "공지" / "연락처" 탭)
────────────────────────────────────────────────────────────────────────────────

  공지사항: 시간표 페이지 하단에 표시. 추가/삭제만 가능 (수정 불가).

  연락처: 직책별 이름·전화번호. 시간표 페이지에 표시.


────────────────────────────────────────────────────────────────────────────────
  5. 데이터 관리 주의사항
────────────────────────────────────────────────────────────────────────────────

  [시즌 초기화]: 해당 시즌의 모든 base_slots, 신청 회차, 신청 데이터 삭제
                 → 복구 불가, 신중하게 사용

  [전체 팀 삭제]: 모든 팀과 연결 데이터 삭제 → 복구 불가

  관리자 비밀번호: 세션 기반 인증. 브라우저 탭을 닫으면 자동 로그아웃.

  Supabase 프리 티어: 7일 이상 아무도 접속하지 않으면 DB가 자동 일시정지됨.
                      첫 접속 시 깨어나는 데 최대 30초 소요.
