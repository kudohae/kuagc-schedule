시간 배정
팀별 동아리방 사용 시간 신청 및 배정

팀 관리 — admin.html "팀" 탭

  • [+ 팀 추가]로 팀 생성: 이름·유형(합주/스쿨/이외)·색상·소개·구성원 입력
  • 구성원 형식: 한 줄에 한 명 — 홍길동 1234 보컬
  • 팀 삭제 시 해당 팀의 모든 슬롯·신청 데이터 함께 삭제됨 주의

회차 운영 — admin.html "시간 배정" 탭

1
[회차 생성] → 오픈/마감 시각 예약 또는 즉시 열기 선택
2
[열기] → "신청 중" 상태로 전환. 팀원들이 1·2·3지망 선택 후 제출
3
마감 후 [배정 실행] → 1지망 → 2지망 → 3지망 순으로 자동 배정 (선착순)
4
결과 확인 후 [초안 승인] → 기존 슬롯이 배정 결과로 교체되어 시간표에 즉시 반영
한 팀이 여러 번 제출하면 가장 최근 제출만 유효합니다.

팀원 요청 처리 — 우측 사이드바

요청 종류승인 시 동작

미사용 신고해당 주에 absent 예외 처리 (시간표에 빗금)

스쿨 신청
수강생 모집 및 반 배정

회차 만들기 — admin.html "스쿨" 탭

1
[+ 새 스쿨 회차 만들기] → 학기·회차 번호 입력 → "초안(draft)" 상태로 생성
2
[+ 반 추가]: 반 이름·담당·정원·요일·시간·설명 입력. 여러 반 추가 가능
3
[열기] 또는 오픈 예약 시각 → "모집중" 상태로 전환
4
[마감하기] 또는 마감 시각 → 자동 마감 + 대기 수강생 일괄 배정

배정 방식

수강생 유형처리 시점방식
신규 수강생신청 즉시1지망 정원 여유 시 배정 → 없으면 2지망 → 없으면 미배정
이전 회차 수강생
(prioritize_returning = true)		마감 후 일괄대기(pending) 상태로 저장 → 마감 시 제출 순서대로 배정
"이전 회차 수강생 대기 처리"는 회차 초안 단계에서 설정해야 합니다.

  • [스쿨 설명 보기] 버튼 → 긴 설명 펼치기/접기
  • 배정 결과: 이름 + 학번 뒷 3자리
🎸 3
합주 신청
곡 신청 → 세션 신청 → DnD 팀 구성. 일반·버스킹 독립 운영

단계 구조

2차 세션 없는 경우 (6단계)
1 준비→ 2 곡 신청→ 3 대기→ 4 세션 신청→ 5 최종 팀 구성→ 6 완료
2차 세션 있는 경우 (8단계)
1 준비→ 2 곡 신청→ 3 대기→ 4 1차 세션→ 5 대기→ 6 2차 세션→ 7 최종 팀→ 8 완료

회차 생성 시 설정 항목

  • 회차 이름 / 최대 곡 수 / 1인 최대 신청 곡 수 / 1인 최대 세션 참여 곡 수
  • 곡 신청 오픈·마감 예약, 세션 신청 오픈·마감 예약
  • 2차 세션 신청 여부 및 오픈·마감 시각
  • 2차 신청 대상: any_song(모든 곡) / missing_only(빈 세션 있는 곡만)

곡 신청 단계 (2단계)

학생이 ensemble.html에서 입력하는 항목:

  • 신청자 성명·학번
  • 곡 제목·아티스트
  • 이 곡에 필요한 세션 전체 (복수 선택)
  • 내가 담당할 세션 — 필요 세션 중에서만 선택 가능
신청자의 세션 담당은 곡 신청 시 자동 등록됩니다. 별도 세션 신청 불필요.

세션 신청 단계 (4/6단계)

  • 확정된 곡 중 원하는 곡 선택 → 신청 가능한 세션만 표시
  • 신청자가 이미 담당한 세션은 선택 불가
  • 1인 최대 참여 곡 수 초과 불가 / 동일 곡 중복 신청 불가

DnD 팀 구성 (5/7단계)

1
[팀 구성 시작] → 모달 열림. 왼쪽: 미배정 풀 / 오른쪽: 곡별 팀
2
카드를 드래그해 곡으로 이동 또는 풀로 되돌리기
3
신청자 배지 카드는 고정(이동 불가). 동일 세션 중복 시 주황 충돌 표시
4
[임시저장] → 새로고침해도 복원됨
5
[확정]: 곡 배치된 인원 → confirmed / 풀에 남은 인원 → rejected(취소선)
6
확정 후 [다음 단계] 버튼으로 진행 (DnD 확정 ≠ 단계 전환)
5단계 → 6단계 전환 시, 1차에서 미확정(pending) 상태로 남은 신청 데이터가 자동 삭제됩니다.

이전 단계 되돌리기

모든 단계에서 [이전 단계] 버튼으로 한 단계씩 되돌릴 수 있습니다.

되돌려도 DB에 저장된 신청 데이터는 삭제되지 않습니다. 필요 시 Supabase에서 직접 정리해야 합니다.

공지사항 / 연락처
시간표 페이지 하단에 표시
  • 공지사항: 추가·삭제만 가능 (수정 불가). 시간표 하단에 표시.
  • 연락처: 직책·이름·전화번호. 시간표 하단에 표시.

데이터 관리 주의사항
기능설명복구
[시즌 초기화]해당 시즌의 모든 base_slots·회차·신청 데이터 삭제불가
[전체 팀 삭제]모든 팀과 연결 데이터 삭제불가
종료 처리(요청)해당 팀의 base_slot 영구 삭제불가
관리자 로그아웃탭/브라우저를 닫으면 자동 로그아웃—
Supabase 프리 티어: 7일 이상 미사용 시 DB가 자동 일시정지됩니다. 첫 접속 시 깨어나는 데 최대 30초 소요. 주 1회 이상 접속하면 방지 가능.
